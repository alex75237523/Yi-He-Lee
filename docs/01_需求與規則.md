# 需求與規則

## 排程

- 程式常駐 Windows 右下角系統匣。
- 每日 `Asia/Taipei 13:35` 自動執行。
- 失敗分為可重試與不可重試；可重試失敗依設定間隔再次執行，達每日上限停止。唯讀、受保護檢視、輸出頁籤受保護與設定錯誤屬不可重試，必須人工修正。
- 手動執行與排程共用單一執行鎖，禁止同時重複執行。

## 網站資料

固定來源：多頭排列與空頭排列；每個來源均抓集中市場及店頭市場完整清單。請求日期、頁面日期、SQLite 交易日期必須等於台北執行當日。網站尚未更新、休市或解析失敗時，不得把前一交易日資料寫成今日資料。

2026-07-09 更新：鉅亨網兩來源僅作多頭／空頭排列完整清單保存與交叉驗證之用，**不再是正式均價來源**。正式均線改為：

- 上市股票：臺灣證券交易所（TWSE）官方每日收盤行情。
- 上櫃股票：證券櫃檯買賣中心（TPEx）官方每日收盤行情。
- MA5／MA20／MA60／MA120：由本系統依上述官方收盤價，依「有效交易日」自行計算（非日曆日、不四捨五入中間值）。

官方來源同樣必須驗證「來源回報的資料日期」等於台北執行當日 `targetDate`，任一來源尚未公布或日期不符時，一律拒絕寫入正式資料，並依重試機制稍後重試；休市日記錄為明確休市狀態，不得用前一交易日資料補齊。詳見 `docs/03_資料庫結構.md`「官方每日收盤價與均線」與 `PROJECT_INSTRUCTIONS_鉅亨技術指標.md`。

## Excel 持股辨識

- 排除頁籤：`總表`、`每日五日均價策略`。
- 不依賴「自持股列表」文字；只要表頭同時找到代號／代碼、股名／名稱、現價即可辨識。（2026-07-11 更正：原「進場價／平均價」為口誤。）
- 表頭包含「出場價」視為已出場清單，不納入持股判斷。
- 預設股名儲存格填滿色為 `#92D050` 時略過。
- 列內出現設定的文字標記時略過。
- 用股票代碼比對技術資料，不以股票名稱作唯一鍵。

## 策略

2026-07-11 更正：比較基準由「進場價／平均價」改為 Excel「現價」欄位（外部 DDE）。下列任一條件成立就產生通知：

```text
MA5 <= 現價
MA20 <= 現價
MA120 <= 現價
```

通知需列出客戶、頁籤、股票代碼、股名、現價、收盤價、MA5、MA20、MA120、實際觸發條件。MA60 只存放及顯示。

### 「現價」欄位（DDE）例外規則

「現價」串接外部 DDE（看盤軟體），不可假設一定是正常數字：

- 儲存格為 Excel 錯誤值（#N/A、#NAME?、#GETTING_DATA 等）、空白、0、負數或無法解析的文字時，該持股不得判斷是否觸發。
- 不得以收盤價、昨日現價或任何其他數值代替現價本身。
- 必須列入「現價異常」清單並寫入 Log，明確告知原因（例如「儲存格為 #N/A（DDE 尚未取得資料，看盤軟體可能未開啟或未連線）」）。
- 注意 Excel 錯誤值經 COM 封送為 `0x800Axxxx` 的 Int32，不可與正常整數價格（例如 2042 元）混淆。
- 2026-07-11 更正：MA5／MA20／MA60／MA120 是本系統依 TWSE／TPEx 官方收盤價自行計算，與「現價」欄位是否連上 DDE 無關；現價異常只代表無法判斷是否觸發，均價本身若已算出，仍必須列在「現價異常」清單與 `每日五日均價策略` 頁籤中，不得留白。

## 顯示與寫入

- 成功或失敗都在螢幕正中央顯示。
- 成功畫面分成「符合均價條件」、「現價異常」及「無法判斷」。
- 結果覆寫至 `每日五日均價策略`，避免重跑重複追加。
- 寫入前以 Excel `SaveCopyAs` 建立包含目前記憶體狀態的備份；寫入後驗證通知筆數；最後呼叫 `Workbook.Save`。
