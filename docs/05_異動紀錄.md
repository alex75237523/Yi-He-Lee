- **均線資料不足時「每日結果」頁籤顯示文字，不再顯示空白**：`MainForm.Results.cs`「符合均價條件」表格的 5／20／120 日均價欄位，先前資料不足（`decimal?` 為 `null`）時顯示空白字串，容易誤判為系統當掉或欄位漏掉；新增 `FormatMa` 顯示「尚未抓到資料」，與既有「歷史收盤價」畫面「資料不足」文字的處理原則一致（不得顯示 0 或空白）。

## 2026-07-09（新增常駐分頁主視窗；移除系統管理員提升權限路徑；修正 Excel 合併儲存格寫入失敗）

實機除錯過程中依序發現並修正三個問題：

- **移除系統管理員提升權限路徑**：需求變更為程式不需要、也不應該以系統管理員權限執行（會與一般權限開啟的 Excel 造成 COM 完整性層級隔離，導致找不到已開啟的活頁簿）。`AdministratorHelper.RestartAsAdministrator()` 與系統匣選單「以系統管理員重新啟動」按鈕已刪除；`TrayApplicationContext.ConfigureAdministratorMenu` 現在只顯示目前權限狀態（一般／系統管理員警告文字），不再提供提升權限的操作路徑。`app.manifest` 原本就是 `asInvoker`，未變更。
- **修正 `Excel 寫入策略結果失敗：無法對合併儲存格執行該動作`（`COMException 0x800A03EC`）**：`ExcelWorkbookService.WriteStrategyResultsCore` 清空輸出頁籤前對 `A1:G{lastClearRow}` 呼叫 `Range.Clear()`，若頁籤內殘留跨出 A:G 欄位邊界的合併儲存格（例如手動編輯留下的舊格式），Excel 會直接拋例外，且每次執行皆會命中、並非偶發。修正為清空前先呼叫 `clearRange.UnMerge()`（無合併儲存格時為無動作，安全）。刻意不判斷 `Range.MergeCells` 屬性，因為範圍內合併狀態不一致時該屬性回傳 `DBNull`，直接轉型會另外拋出例外。
- **新增常駐分頁主視窗，取代 Modal 設定視窗與獨立結果彈出視窗**：新增 `Forms/MainForm.cs`（＋`MainForm.Operations.cs`／`MainForm.Settings.cs`／`MainForm.Results.cs` 三個 partial 檔案），內含「操作」「設定」「每日結果」三個頁籤：
  - 雙擊系統匣圖示、選單「設定」皆改為開啟／喚醒同一個常駐 `MainForm`（單例重用，比照既有 `HistoricalPriceForm` 模式），不再彈出 Modal 對話框。
  - 「設定」頁籤內容搬自原 `SettingsForm`（一般設定／資料來源網址兩個內層頁籤），改為「儲存設定」按鈕即時儲存，不再需要關閉視窗才生效；儲存邏輯（`_settingsStore.SaveAsync` ＋開機啟動設定）維持在 `TrayApplicationContext`，Form 只負責畫面與驗證，符合既有分層原則。
  - 「每日結果」頁籤內容搬自原 `ResultCenterForm`（成功／失敗兩種版面、均線觸發與無法判斷清單），執行完成或失敗時（`ShowSuccessCenter`／`ShowFailureCenter`）自動將主視窗帶到前景並切換到此頁籤，不再另開視窗。
  - 主視窗關閉（按 X）只隱藏、不結束程式，行為與原本「關閉結果視窗後程式仍留在系統匣」一致；只有真正結束程式時才會關閉。
  - 已刪除 `Forms/SettingsForm.cs`、`Forms/ResultCenterForm.cs`（內容已搬入 `MainForm.*.cs`，避免同一套 UI 邏輯留兩份）。
- **測試**：`dotnet build`（整個方案）與 `dotnet test`（128／128）皆成功；WinForms 這幾個 Form 目前沒有專屬單元測試，維持現狀。

## 2026-07-09（修正 Excel 已開啟仍判定找不到活頁簿）

依實機 Log 與第 16 次執行畫面追查，故障分成兩階段：前段曾因執行目錄缺少 `office, Version=16.0.0.0` 而無法進入 Excel Interop；該相依檔補齊後，現行錯誤轉為 `ExcelRunningObjectResolver` 找不到指定活頁簿。原解析器只使用 ROT 與單一 `GetActiveObject`，且在取得 ROT 物件前先以 Moniker 顯示名稱過濾；多個 Excel 執行個體、活頁簿尚未註冊 ROT、Moniker 不含完整路徑、受保護檢視或權限隔離時，都會被合併成同一個「找不到」訊息。此外，此類錯誤被一般例外直接包裝，沒有真的執行設定中的 Excel 短暫重試。

- `ExcelRunningObjectResolver`：取消 ROT 顯示名稱的前置過濾，實際檢查 ROT 中的 Workbook／Application／Window；保留 `GetActiveObject`，並新增巡覽所有 `XLMAIN`／`EXCEL7` 視窗後透過 `AccessibleObjectFromWindow(OBJID_NATIVEOM)` 取得每一個 Excel 執行個體，支援多執行個體及尚未正常登錄 ROT 的情況。
- 新增 `ExcelWorkbookPathMatcher`：集中正規化本機路徑與 `file://` URI，仍以完整路徑為唯一安全命中條件；若只找到同名但不同路徑的活頁簿，明確要求重新選檔，不會只憑檔名寫入錯誤副本。
- 新增受保護檢視辨識：檢查 `ProtectedViewWindows.SourcePath／SourceName`；命中指定檔案時回報「請按啟用編輯」，並依規範分類為不可重試。
