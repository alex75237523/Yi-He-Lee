# 測試與驗收

## 自動測試

- MA5、MA20、MA120 任一成立即通知。
- 三項都不成立時不通知。
- 技術資料缺少時列為無法判斷。
- 綠色 `#92D050` 與文字標記排除。
- 一般黃色持股不可誤排除。
- 固定來源及 13:35 不可被關閉。
- 鉅亨表格完整解析、缺表頭失敗、頁面日期擷取。

### 官方每日收盤價與均線（2026-07-09 新增，共 68 個自動測試全數通過）

- TWSE／TPEx 正常回應解析（含收盤價含逗號、空白、`--`、`N/A` 之解析與略過）。
- TWSE 休市日明確查無資料；TPEx 靜默回傳前一交易日資料時，Service 層可正確偵測日期不符並拒絕寫入。
- HTTP 200 但為錯誤頁／非 JSON／缺少必要欄位時拋出可重試例外，不視為成功。
- 民國年／西元年、斜線格式／緊湊格式互轉。
- 週末不呼叫官方來源直接記錄休市；平日 TWSE 明確休市時，TPEx 日期不符一併記為休市而非「尚未公布」。
- 同一目標日期重跑不重複呼叫官方來源（冪等略過）。
- 歷史回補只查詢 `targetDate` 以前的日期，不得改寫成執行當日。
- MA5／MA20／MA60／MA120 以固定可人工驗算的收盤價驗證；恰好 120／119 個有效交易日之邊界測試；資料不足時對應均線為 `null` 且不誤將不足天數硬算。
- SQLite 層：同日重跑同一股票只更新不新增重複列；批次含不同交易日整批拒絕；交易中途失敗完整回滾不留下半批資料；依交易日期排序並受最大天數限制。
- 少量 Integration Test（`Category=Integration`）驗證 TWSE／TPEx 目前官方端點與欄位仍可使用；無網路環境可用 `dotnet test --filter Category!=Integration` 排除，執行失敗會明確標示為連線問題而非假裝成功。

### 逐檔歷史完整性、股票代碼正規化與 Excel 表格邊界（2026-07-11 新增，共 211 個自動測試全數通過，另有 4 個 `Category=Integration` 真實連線測試）

- **均線計算邊界**：4／5／19／20／59／60／119／120 筆有效交易日各邊界案例；最新資料日期不等於策略日期時狀態為 `TodayCloseMissing`（與「歷史資料不足」明確區分）；重複交易日不得重複計入平均（防禦性去重）。
- **逐檔歷史回補**：市場整體交易日數已足夠但指定持股仍不足時必須繼續回補；同市場 A 股票足夠、B 股票不足時仍會繼續抓取該市場；未指定持股代碼時維持既有「僅檢查市場整體」行為（向下相容）；回補過程官方來源發生例外時該市場批次記為 `Failed`，供 `DailyJobService` 產生逐檔「歷史回補失敗」狀態；歷史回補一律不得把前一交易日寫成 targetDate。
- **股票代碼格式與辨識**（`StockIdentityResolverTests`）：一般股票／4碼與5碼ETF／槓桿反向／債券／主動式ETF／DR 格式有效並納入策略；權證格式有效但明確排除於策略之外並留下原因；7～8 位數金額（`10037677`、`13818762` 等）格式無效不得視為股票代碼。
- **股票代碼官方主檔解析與補零**（`StockIdentityResolutionServiceTests`）：`0050` 直接命中官方主檔；`50` 經官方主檔確認 `0050` 存在時解析為 `0050`；官方主檔查無補零候選時不得盲目補零；`00982A` 保留原樣。
- **Excel 表格邊界辨識**（`ExcelWorkbookServiceHeaderDetectionTests`，不啟動 Excel COM）：「現貨現價」與「現價」皆可識別為同一欄位；遇到日期／權益數／保證金／小計等其他表格表頭時上一個持股區塊必須結束；遇到新持股表頭／已出場表頭時上一個區塊必須結束；股票代碼候選過濾規則；表頭正規化。
- **均線策略診斷欄位**（`StrategyEvaluationServiceTests`）：股票代碼無法識別／權證排除於策略之外時產生診斷通知；均線觸發時附上計算狀態「正常」；逐檔歷史資料不足但部分均線（例如僅 MA5）已觸發時仍產生通知並標示「歷史資料不足」原因，不得因部分均線空白而略過通知或隱藏原因。
- **資料庫 Migration**（`SqliteMarketDataRepositoryMigrationTests`）：`StockMovingAverage` 舊資料庫缺少 `LatestAvailableTradeDate`／`MissingReason` 欄位時自動安全補上且保留既有資料列；遷移後可正常寫入並讀回。

### 歷史收盤價查詢／使用者手動回補（2026-07-09 新增，共 115 個自動測試全數通過，含 3 個 `Category=Integration` 真實連線測試）

- `TransientFailureClassifierTests`：HTTP 408／429／5xx 與網路逾時視為暫時性；HTTP 4xx（非408/429）、HTML／JSON 結構變更、驗證頁等訊息不得盲目重試；無法辨識訊息預設暫時性但仍受重試次數上限約束。
- `StockHistoryImportServiceTests`：工作數＝有效交易日數×市場數；候選日期排除週六週日且不含當日；市場範圍篩選正確；`MaxConcurrency` 確實生效（以計數器量測觀察到的最大並行數）；暫時性錯誤依指數退避重試並正確記錄重試次數；**`IMarketPriceService` 以 `Failed`／`NotPublished` 摘要（而非例外）回報暫時性錯誤時仍會正確重試**（實機串接官方端點時發現：`MarketPriceService` 對 HTTP／解析失敗一律內部捕捉並轉為狀態摘要回傳、不拋例外，若重試邏輯只攔截例外，暫時性錯誤永遠不會被重試，已修正並補上對應測試）；`NotPublished` 重試耗盡時最終狀態為「等待來源更新」而非「失敗」；非暫時性錯誤不重試直接失敗；部分工作失敗時整批狀態為「部分失敗」而非卡在執行中；取消後批次與尚未執行工作皆標記已取消；重新整理（重新建立 Repository 執行個體）後仍可讀取最新批次進度。
- `SqliteStockPriceImportRepositoryTests`：批次建立時工作初始狀態皆為排隊中；同批次相同市場＋日期重複違反唯一鍵；工作完成後批次進度依明細彙總且百分比正確；全部成功／全部失敗／部分失敗時最終批次狀態正確；取消批次後尚在排隊或執行中的工作皆標記已取消；取得最新批次回傳最近建立的一筆。
- `SqliteMarketDataRepositoryQueryTests`：SQL Window Function 計算之 MA5／20／60／120 與人工驗算（及既有 `MovingAverageService` Rolling Sum 邏輯）結果一致；資料不足時 MA 欄位為 `null` 而非 0；市場別／代碼或名稱關鍵字／日期區間篩選正確；分頁不重複且不會一次回傳全部資料；`GetLatestTradeDateAsync` 於無資料時回傳 `null`。
- `CnyesStockPriceValidationServiceTests`：鉅亨頁面日期須等於目標交易日期才比對，否則標記 `SourceDateMismatch`；股票未出現在多頭／空頭清單時標記 `NotApplicable`；差異 ≤0.01 為 `Matched`，>0.01 為 `Mismatched`；僅回補5日資料時 MA20／60／120 標記 `InsufficientHistory`、MA5 仍正常比對；鉅亨網本次無資料時標記 `SourceUnavailable` 且不影響官方資料。
- `StockHistoryImportIntegrationTests`（`Category=Integration`，真實連線）：對最近1個有效交易日、上市＋上櫃兩個市場，實際呼叫 TWSE／TPEx 官方端點並寫入暫存 SQLite，驗證整條並行回補路徑（排程、重試、Repository 進度、Upsert）可用真實網路運作。實測執行時（2026-07-09，請求日期 2026-07-08）：TWSE 成功寫入 1361 筆、交易日期與請求日期相符；TPEx 該次查詢意外持續回傳「2026-07-09」而非請求的「2026-07-08」（與既有註解「TPEx 對非交易日會回退到較早日期」的方向相反，屬本次測試發現的新行為樣態），既有 `MarketPriceService` 日期比對邏輯正確偵測不符並拒絕寫入，本功能之重試機制依設定重試3次後正確記為「等待來源更新」，全程未寫入任何日期不符資料，符合「絕不得把非目標日期資料當成功」的規則。

## Windows 人工驗收

1. Excel 未開啟：應中央顯示失敗，Log 有明確原因，排程可重試。
2. Excel 正在編輯儲存格：短重試後成功，或達上限顯示失敗。
3. Excel 有 Modal 對話框：不得卡死；關閉後重新執行可成功。
4. 活頁簿唯讀／受保護檢視：不得寫入。
5. 輸出頁籤受保護：不得寫入，原檔不得被破壞。
6. 頁面顯示昨日日期：SQLite 正式技術表不得出現今日資料。
7. 同一天重跑：技術資料、持股快照、通知不得重複。
8. 綠色股名列：不得進入判斷清單。
9. 沒有「自持股」標題但表頭完整的區塊：必須納入。
10. 含「出場價」的已出場表格：不得納入。
11. 中央通知：需在螢幕中央，完整顯示客戶、股票及觸發均價。
12. 系統匣：關閉中央視窗後程式仍在右下角運行。

### 歷史收盤價查詢／使用者手動回補（2026-07-09 新增）

13. 系統匣選單點選「歷史收盤價」：開啟查詢畫面，市場別預設「全部」、交易日數預設「5日」。
14. 按下「立即回補」：立即顯示批次進度（不須等待整批完成才有畫面回應），可看到約 10 個工作明細（5 個交易日 × 2 個市場）陸續由排隊中轉為執行中、成功。
15. 關閉畫面後重新開啟「歷史收盤價」：仍可看到上一批進度（由資料庫回復，非僅存於記憶體）。
16. 執行中按下「取消抓取」：批次與尚未執行的工作狀態變為「已取消」，已完成的工作維持原狀態不受影響。
17. 回補完成後查詢：有 5 筆有效收盤價的股票顯示 MA5 數值；MA20／MA60／MA120 顯示「資料不足」文字，不得顯示 0。
18. 重複執行「立即回補」（同樣日期範圍）：不會產生重複的 `StockDailyPrice` 列（冪等）。
19. 切換市場別／輸入股票代碼或名稱關鍵字／設定日期區間：查詢結果正確篩選且分頁正常（不會一次載入全部資料、頁籤切換順暢）。
20. 按下「與鉅亨網交叉驗證」：顯示相符／差異／資料不足／不適用／日期不符的統計彈窗，且不影響已查詢到的官方歷史收盤價。
