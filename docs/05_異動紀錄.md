# 異動紀錄

## 2026-07-11（修正：逐檔歷史完整性驗證、股票代碼正規化與 Excel 表格邊界辨識）

分支：`fix/per-stock-moving-average-history`。修正「每日五日均價策略」中大量股票只有 MA5、缺少 MA20／MA60／MA120，以及非股票資料被誤判的問題。

### 根因

1. **歷史回補以市場整體交易日數判斷是否足夠**：`MarketPriceService.BackfillHistoryAsync` 原本只檢查
   `GetDistinctTradeDateCountAsync(marketType)`（該市場「任一股票」湊出的相異交易日數），只要市場整體達
   120 日就停止回補；但個別股票（例如上櫃 5351 鈺創、8299 群聯、3691 碩禾）可能因掛牌較晚、曾停牌或
   回補當時尚無資料，實際筆數遠低於 120。市場整體足夠不代表每一檔股票都足夠。
2. **股票代碼正規表示式過寬**：`^[0-9A-Z]{4,10}$` 會把 Excel 中的權益數、保證金等 7～8 位數金額
   （例如 `10037677`）誤判為股票代碼。
3. **ETF 前導零遺失時無回復機制**：Excel 儲存格若被當成數字讀取，`0050` 會讀成 `50`；原程式沒有任何
   補零機制，也沒有官方主檔比對，這類持股會直接判斷失敗或被過寬正規表示式誤收。
4. **Excel 表格邊界辨識只認「現價」**：無法識別「現貨現價」等同義表頭；且只有遇到「新的持股表頭」或
   「出場價表頭」才會結束區塊，遇到「日期／權益數／保證金／小計」等其他彙總表格時不會停止，會把該表格
   內容誤判為持股列延伸掃描。

### 修正方式

- **`MarketPriceService.BackfillHistoryAsync`**：新增可選參數 `listedStockCodes`／`otcStockCodes`
  （既有 `emergingStockCodes` 沿用），完成判斷改為「市場整體交易日數足夠」**且**「傳入的持股逐檔皆已
  達 `RequiredTradingDaysForMa120`」兩者皆成立才停止回補；任一持股逐檔仍不足，即使市場整體已足夠也會
  繼續回補該市場（回補仍是整批市場下載，效率不變，只有「完成判斷」改為逐檔）。未傳入代碼時維持既有
  「僅檢查市場整體」行為，向下相容。`IMarketPriceService.BackfillHistoryAsync`、`HistoricalBackfillJob.RunAsync`
  同步新增對應參數並轉發。
- **`DailyJobService`**：讀取 Excel 持股後，改由新增的 `StockIdentityResolutionService` 解析每一筆
  持股代碼，取得逐檔市場別（`listedStockCodes`／`otcStockCodes`／`emergingStockCodes`）後才呼叫回補；
  回補批次若出現 `Failed` 狀態，均線計算後會把該市場仍 `InsufficientHistory` 的股票升級為
  `CalculationStatus.BackfillFailed` 並附上原因，不再與「單純歷史資料不足」混為一談。
- **`StockCodeNormalizer`／`StockIdentityResolver`（新增，`YiHeLee.Domain/StockIdentity.cs`）**：
  取代過去「4～10 碼英數字」的過寬規則，改依台股實際商品格式分類：
  - 4 碼數字＝一般股票／興櫃／4碼ETF（0050、0056 依 `00` 開頭標記為 ETF）。
  - `00` 開頭 5 碼數字＋選用單一英文字尾＝ETF：無字尾為一般 ETF、`L／R`＝槓桿反向、`B`＝債券 ETF、
    其他單一字母＝主動式 ETF（`00631L`、`00982A`、`00679B`）。
  - `91` 開頭 6 碼數字＝存託憑證 DR。
  - 其餘 6 碼數字＝權證，格式有效但**明確排除於均線策略之外**（短期衍生性商品，價格行為與長期均線
    策略無關），並留下原因。
  - 其餘（含 7～8 位數金額）一律視為格式無效，不得判斷為股票代碼。
  `StockCodeNormalizer.Normalize` 為全系統唯一正規化規則來源（去空白、轉大寫），
  `StrategyEvaluationService.NormalizeStockCode`、`ExcelWorkbookService` 的股票代碼正規化皆改為委派
  給它，不再各自實作。
- **`StockIdentityResolutionService`（新增，`YiHeLee.Application/Services/`）**：協調正規化、前導零
  回復與官方主檔驗證。先以原始代碼查 `IMarketDataRepository.GetStockMarketTypesAsync`（官方主檔，
  今日已出現在 TWSE／TPEx／TPEx興櫃行情中的代碼）；查無資料且代碼為 1～3 碼純數字時，嘗試補零為 4 碼
  或 5 碼後再查一次官方主檔，**只有補零後的代碼確實存在於官方主檔才採用**，不會盲目補零（例如
  `50`→`0050` 需官方主檔已有 `0050` 才會採用；若官方主檔完全查無對應候選，仍標示為無法識別）。
- **`ExcelWorkbookService`**：
  - 股票代碼候選過濾改用 `StockIdentityResolver.Resolve(...).IsFormatValid`（涵蓋格式合法性），
    另外保留 1～3 碼純數字候選交由 `StockIdentityResolutionService` 以官方主檔補零確認，不在 Excel
    讀取階段就用字串長度武斷判定。
  - 表頭同義字新增「現貨現價」（`CurrentPriceHeaderSynonyms`），與「現價」視為同一欄位。
  - 新增「其他表格表頭」邊界偵測（`OtherTableHeaderKeywords`：日期／權益數／保證金／小計／總計／合計，
    一列出現任兩個以上即視為其他表格表頭），持股區塊結束邊界改採「下一個持股表頭」與「下一個其他表格
    表頭」兩者較早者，不再只看下一個持股表頭。
  - 股票名稱空白時不建立持股；同一區塊內連續兩列代碼與名稱皆空白視為區塊已結束，停止該區塊掃描。
  - `FindActiveHoldingHeaders`／`IsAcceptableStockCodeCandidate`／`NormalizeHeader`／
    `CellValueAccessor`／`HeaderDefinition`／`TableHeaderCandidate` 存取層級由 `private` 改為
    `internal`，供測試專案（既有 `InternalsVisibleTo`）直接驗證表格邊界邏輯，不需啟動 Excel COM。
- **`MovingAverageResult`／`CalculationStatus`（Domain 擴充）**：新增 `LatestAvailableTradeDate`／
  `MissingReason` 欄位與 `HasMa5／20／60／120`、`MissingTradingDaysForMa120` 計算屬性；
  `CalculationStatus` 新增 `TodayCloseMissing`（當日收盤價尚未取得，取代過去與「歷史資料不足」混用的
  舊行為）與 `BackfillFailed`（回補過程發生錯誤導致缺口未補齊）。`MovingAverageService.Calculate`
  同步防禦性去重同一交易日重複資料（避免上游意外提供重複列時被重複計入平均）。
- **`StrategyEvaluationService.Evaluate`**：新增可選參數 `resolutions`；代碼無法識別或非策略商品
  （權證）時，優先於現價／均線判斷直接產生診斷通知（沿用既有 `AlertKind.TechnicalIndicatorMissing`
  歸類於「無法判斷」，不新增 AlertKind 避免影響既有 UI 分頁邏輯與資料庫 CHECK）；`StrategyAlert`
  新增 `DiagnosticStatus`／`MissingReason`／`AvailableTradingDayCount`／`LatestAvailableTradeDate`
  診斷欄位，涵蓋「正常／歷史資料不足／當日收盤價缺失／股票代碼無法識別／非策略商品／Excel現價異常／
  歷史回補失敗」七種狀態；均線部分為 null 但已算出的均線仍觸發條件時，維持通知並如實標示原因，不再
  把「部分均線空白」當成沒有說明的正常結果。
- **Excel 輸出（`每日五日均價策略`）欄位由 7 欄擴充為 14 欄**：代碼／名稱／市場別／收盤價／5日均價／
  20日均價／60日均價／120日均價／有效交易日數／最新收盤日期／計算狀態／缺少原因／來源頁籤／原始列號。
  來源頁籤／原始列號沿用既有 `SheetName`／`ExcelRow`，不重複新增欄位。
- **資料庫**：`StockMovingAverage` 新增 `LatestAvailableTradeDate`（TEXT NULL）、`MissingReason`
  （TEXT NULL）兩個選填欄位，以既有 `PRAGMA table_info` 檢查後 `ALTER TABLE ADD COLUMN` 安全新增，
  不刪除或覆蓋既有資料列；`database/schema.sql` 與內建 `SchemaSql` 同步更新。
