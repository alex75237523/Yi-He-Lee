# 架構與資料流程

```text
DailyScheduleCoordinator
  -> DailyJobService
     -> CrawlerRegistry -> CnyesTechnicalAlignmentCrawler -> Parser
     -> SqliteYiHeLeeRepository（鉅亨多頭／空頭清單，四個必要批次驗證後單一交易；僅作清單保存與交叉驗證）
     -> DailyMarketDataJob -> MarketPriceService -> ITwseMarketDataProvider／ITpexMarketDataProvider
        -> SqliteMarketDataRepository（官方每日收盤價，來源日期驗證後單一交易 Upsert）
     -> HistoricalBackfillJob -> MarketPriceService（best-effort 回補 MA120 所需歷史資料，保留原始 TradeDate）
     -> ExcelWorkbookService（讀取持股）
     -> MovingAverageService -> SqliteMarketDataRepository（依官方收盤價計算並快取 MA5／20／60／120）
     -> StrategyEvaluationService（正式判斷改用官方均線；鉅亨資料僅做輕量交叉驗證並記錄於 Log）
     -> SqliteYiHeLeeRepository（持股快照及通知）
     -> ExcelWorkbookService（覆寫輸出頁籤並儲存，新增「資料來源」「計算時間」欄位）
     -> WinFormsUserInteraction -> 中央通知視窗
```

## 官方每日收盤價來源分層

- `ITwseMarketDataProvider`／`TwseMarketDataProvider`、`ITpexMarketDataProvider`／`TpexMarketDataProvider`：只呼叫官方端點、轉成 DTO、回報來源資料日期與解析錯誤，不含日期驗證或 SQL。
- `IMarketPriceService`／`MarketPriceService`：台北日期與來源日期驗證、休市／尚未公布判斷、冪等 Upsert、批次狀態記錄、歷史回補協調。
- `IMovingAverageService`／`MovingAverageService`：依有效交易日計算 MA5／MA20／MA60／MA120，資料不足時對應欄位為 `null`。
- `DailyMarketDataJob`／`HistoricalBackfillJob`：排程觸發用的薄封裝，只呼叫 `IMarketPriceService`，不含 HTTP、HTML／JSON 解析或 SQL。
- `IMarketDataRepository`／`SqliteMarketDataRepository`：`StockMaster`／`StockDailyPrice`／`StockMovingAverage`／`OfficialPriceBatch` 的參數化 SQL、交易與 Upsert。

## 重試層級

- 爬蟲短重試：網路、瀏覽器、DOM 暫時錯誤。
- Excel 短重試：RPC 呼叫被拒、Excel 忙碌、正在編輯儲存格。
- 排程長重試：網站尚未更新當日資料、短重試耗盡等可重試狀態。
- 設定無效、必要 Provider 不存在、Excel 唯讀／受保護檢視、輸出頁籤受保護等不可重試錯誤，需要人工修正。
- 非必要來源必須集中／店頭兩個市場都成功才會納入正式批次；其中一個市場失敗時整個非必要來源略過。
- 官方每日收盤價（TWSE／TPEx）：來源資料日期不等於 `targetDate` 時記為 `NotPublished`（可重試，等同「網站尚未更新」）；TWSE 明確回報無資料時記為 `Holiday`。TPEx 端點對非交易日會靜默回傳最近一個交易日資料而非報錯，因此僅在同日 TWSE 已確認休市時，才會把 TPEx 的日期不符情形也記為 `Holiday`，否則一律視為 `NotPublished`，絕不接受用前一交易日資料頂替當日。

## 權限

預設與使用者同權限執行，以連接一般權限的 Excel。可從系統匣選單要求 UAC 提升，但提升後 Excel 也必須以相同權限啟動。
