# 異動紀錄

## 2026-07-09（正式均價來源改為 TWSE／TPEx 官方資料）

- 新增官方每日收盤價來源：`ITwseMarketDataProvider`／`TwseMarketDataProvider`、`ITpexMarketDataProvider`／`TpexMarketDataProvider`（`src/YiHeLee.Infrastructure/MarketData/`）。實際呼叫並驗證官方 Swagger 與端點後採用：
  - TWSE：`https://www.twse.com.tw/exchangeReport/MI_INDEX?response=json&date={yyyyMMdd}&type=ALLBUT0999`（可指定日期、回應含明確 `date` 欄位；非交易日回傳明確「查無資料」）。
  - TPEx：`https://www.tpex.org.tw/web/stock/aftertrading/daily_close_quotes/stk_quote_result.php?l=zh-tw&d={民國年/月/日}&s=0,asc,0`（可指定日期）。**實測發現**此端點對非交易日會靜默回傳最近一個交易日資料而非報錯，因此程式一律嚴格比對回應日期與 `targetDate`，不符時拒絕寫入並記錄為 `NotPublished`（除非同日 TWSE 已確認休市才記為 `Holiday`）。
- 新增 `IMarketPriceService`／`MarketPriceService`：來源日期驗證、休市／尚未公布判斷、冪等 Upsert、批次狀態記錄。
- 新增 `IMovingAverageService`／`MovingAverageService`：依有效交易日計算 MA5／MA20／MA60／MA120，中間不四捨五入，資料不足時對應欄位為 `null` 並標記 `CalculationStatus=InsufficientHistory`。
- 新增 `DailyMarketDataJob`／`HistoricalBackfillJob`：排程觸發用薄封裝，只呼叫 `IMarketPriceService`，不含 HTTP、解析或 SQL。
- 新增 `IMarketDataRepository`／`SqliteMarketDataRepository`：`StockMaster`／`StockDailyPrice`／`StockMovingAverage`／`OfficialPriceBatch` 四張新資料表，全部參數化 SQL、交易與 Upsert，`decimal` 全程搭配既有安全 SQLite Decimal Mapping（原生 `AddWithValue(decimal)` + `GetDecimal()`）。
- `StrategyEvaluationService.Evaluate` 改為消費 `MovingAverageResult`（官方均線）而非鉅亨網 `TechnicalIndicator`；鉅亨網多頭／空頭排列清單改為僅供清單保存，並在 `DailyJobService` 新增輕量交叉驗證（官方 MA20 與鉅亨網 MA20 差異超出容忍範圍時記錄 Log 警告，不影響正式判斷）。此舉同時消除先前已知風險「同一股票同時出現在多頭與空頭清單時靜默挑一筆」對正式判斷的影響（正式判斷已不再依賴鉅亨網數值）。
- `DailyJobService.RunAsync` 新增官方價格批次驗證：兩來源皆休市時記為 `JobStatus.NoTradingData` 並正常結束；任一來源尚未公布時拋出既有 `WebsiteNotUpdatedException`（可重試）；官方價格成功後才計算均線、判斷策略、寫入 Excel。
- `ExcelWorkbookService` 輸出頁籤新增「資料來源」「計算時間」兩欄（欄位由 17 欄擴充為 19 欄，A1:Q 範圍全數改為 A1:S），保留既有備份、驗證筆數、原子寫入流程不變。
- `AppSettings` 新增 `OfficialMarketDataSettings`：官方端點樣板、HTTP Timeout／短重試、MA120 所需交易日數、歷史回補最大回看日曆日數與節流間隔，皆可設定但不得關閉日期驗證規則。
- 新增 68 個自動測試（含 2 個標記 `Category=Integration` 的真實連線測試，已驗證 TWSE／TPEx 端點於本次開發時仍可正常使用）；詳見 `docs/04_測試與驗收.md`。
- 已知限制：鉅亨網與官方 MA 的交叉驗證僅記錄於程式 Log，未另建持久化差異資料表；後續如需歷史查詢差異，可再擴充 `OfficialPriceBatch` 或新增比對表。

## 2026-07-09（Claude Code 接手：Build 修復與可攜式路徑）

首次在具備 .NET 8 SDK 的環境實際執行 `dotnet restore／build／test`，修正三類根因後 Debug／Release Build 與全部 14 個自動測試皆成功：

- `src/YiHeLee.Infrastructure/Excel/ExcelWorkbookService.cs`、`ExcelRunningObjectResolver.cs`：命名空間 `YiHeLee.Infrastructure.Excel` 與 `using Excel = Microsoft.Office.Interop.Excel;` 別名同名，導致編譯器將 `Excel.Workbook` 等型別解析回自家空白命名空間而報錯（CS0234）。改用別名 `ExcelInterop` 取代 `Excel`，不變更任何邏輯。
- `src/YiHeLee.Infrastructure/Crawlers/CnyesTechnicalAlignmentCrawler.cs`：`YiHeLee.Application.Abstractions.IClock` 與 `Microsoft.Playwright.IClock` 同名造成 CS0104 模稜兩可參考，新增明確別名 `using IClock = YiHeLee.Application.Abstractions.IClock;` 解決。
- `src/YiHeLee.App/Program.cs`：命名空間 `YiHeLee.Application` 與 WinForms 的 `System.Windows.Forms.Application` 類別同名，`Application.Run(context)` 被誤解析為命名空間（CS0234），改為完整名稱 `System.Windows.Forms.Application.Run(context)`。
- `tests/YiHeLee.Tests/CnyesTechnicalAlignmentParserTests.cs`：`new List<string[]> { [...], [...] }` 語法中，`{ }` 集合初始化式內以 `[` 開頭的元素會被剖析器誤判為索引初始設定式（`this[key] = value`），造成 CS1003／CS1525。改為整個集合改用 C# 12 集合運算式 `List<string[]> rows = [ [...], [...] ];`，語意不變。
- `src/YiHeLee.App/AppPaths.cs`：依 `docs/07_可攜式SQLite規格.md`，將根目錄由 `%LOCALAPPDATA%\Yi He Lee` 改為 `AppContext.BaseDirectory`（程式所在目錄），並在建立 Data／Logs／Backups 資料夾失敗時，明確提示路徑與權限原因，避免使用者放在 `Program Files` 等不可寫目錄時得到不清楚的例外。同步更新 `README.md` 資料位置說明。
