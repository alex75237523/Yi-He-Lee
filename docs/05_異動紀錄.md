## 2026-07-09（Claude Code 接手：Build 修復與可攜式路徑）

首次在具備 .NET 8 SDK 的環境實際執行 `dotnet restore／build／test`，修正三類根因後 Debug／Release Build 與全部 14 個自動測試皆成功：

- `src/YiHeLee.Infrastructure/Excel/ExcelWorkbookService.cs`、`ExcelRunningObjectResolver.cs`：命名空間 `YiHeLee.Infrastructure.Excel` 與 `using Excel = Microsoft.Office.Interop.Excel;` 別名同名，導致編譯器將 `Excel.Workbook` 等型別解析回自家空白命名空間而報錯（CS0234）。改用別名 `ExcelInterop` 取代 `Excel`，不變更任何邏輯。
- `src/YiHeLee.Infrastructure/Crawlers/CnyesTechnicalAlignmentCrawler.cs`：`YiHeLee.Application.Abstractions.IClock` 與 `Microsoft.Playwright.IClock` 同名造成 CS0104 模稜兩可參考，新增明確別名 `using IClock = YiHeLee.Application.Abstractions.IClock;` 解決。
- `src/YiHeLee.App/Program.cs`：命名空間 `YiHeLee.Application` 與 WinForms 的 `System.Windows.Forms.Application` 類別同名，`Application.Run(context)` 被誤解析為命名空間（CS0234），改為完整名稱 `System.Windows.Forms.Application.Run(context)`。
- `tests/YiHeLee.Tests/CnyesTechnicalAlignmentParserTests.cs`：`new List<string[]> { [...], [...] }` 語法中，`{ }` 集合初始化式內以 `[` 開頭的元素會被剖析器誤判為索引初始設定式（`this[key] = value`），造成 CS1003／CS1525。改為整個集合改用 C# 12 集合運算式 `List<string[]> rows = [ [...], [...] ];`，語意不變。
- `src/YiHeLee.App/AppPaths.cs`：依 `docs/07_可攜式SQLite規格.md`，將根目錄由 `%LOCALAPPDATA%\Yi He Lee` 改為 `AppContext.BaseDirectory`（程式所在目錄），並在建立 Data／Logs／Backups 資料夾失敗時，明確提示路徑與權限原因，避免使用者放在 `Program Files` 等不可寫目錄時得到不清楚的例外。同步更新 `README.md` 資料位置說明。
