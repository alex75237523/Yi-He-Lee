# 需求與規則

## 排程

- 程式常駐 Windows 右下角系統匣。
- 每日 `Asia/Taipei 13:35` 自動執行。
- 失敗分為可重試與不可重試；可重試失敗依設定間隔再次執行，達每日上限停止。唯讀、受保護檢視、輸出頁籤受保護與設定錯誤屬不可重試，必須人工修正。
- 手動執行與排程共用單一執行鎖，禁止同時重複執行。

## 網站資料

固定來源：多頭排列與空頭排列；每個來源均抓集中市場及店頭市場完整清單。請求日期、頁面日期、SQLite 交易日期必須等於台北執行當日。網站尚未更新、休市或解析失敗時，不得把前一交易日資料寫成今日資料。

2026-07-09 更新：鉅亨網兩來源僅作多頭／空頭排列完整清單保存與交叉驗證之用，**不再是正式均價來源**。正式均線改為：

- 上市股票：臺灣證券交易所（TWSE）官方每日收盤行情。
- 上櫃股票：證券櫃檯買賣中心（TPEx）官方每日收盤行情。
- MA5／MA20／MA60／MA120：由本系統依上述官方收盤價，依「有效交易日」自行計算（非日曆日、不四捨五入中間值）。

官方來源同樣必須驗證「來源回報的資料日期」等於台北執行當日 `targetDate`，任一來源尚未公布或日期不符時，一律拒絕寫入正式資料，並依重試機制稍後重試；休市日記錄為明確休市狀態，不得用前一交易日資料補齊。詳見 `docs/03_資料庫結構.md`「官方每日收盤價與均線」與 `PROJECT_INSTRUCTIONS_鉅亨技術指標.md`。

## Excel 持股辨識

- 排除頁籤：`總表`、`每日五日均價策略`。
- 不依賴「自持股列表」文字；表頭必須同時找到代號／代碼、股名／名稱、進場價/平均價、現價，四者缺一不可才視為有效持股表。
- 「現價」表頭可接受同義字「現貨現價」（2026-07-09 新增，經實際客戶工作簿確認為同一欄位；新增其他同義字前必須再次確認，避免誤認不相關欄位）。
- 「進場價/平均價」表頭可接受全形／半形斜線（`進場價/平均價`、`進場價／平均價`），`NormalizeHeader` 會統一正規化為半形斜線後比對。

> **2026-07-11 需求異動更正說明**：本文件曾在 2026-07-11 記載「原『進場價／平均價』為口誤，正確欄位為『現價』」，並將比較基準改為單一欄位「現價」。**此更正本身是錯誤的**：客戶實際工作簿確實同時存在「進場價/平均價」與「現價」兩個欄位，並非同一欄位的誤植。使用者已於本次（2026-07-11 稍後）正式恢復雙價格判斷需求，本文件下方所有「比較基準」相關段落均以本次恢復後的規則為準；先前「原進場價／平均價為口誤」的說法作廢，不再適用。
- 表頭包含「出場價」視為已出場清單，不納入持股判斷。
- 2026-07-11 新增：遇到「日期／權益數／保證金／小計／總計／合計」等其他表格表頭（一列出現任兩個以上關鍵字）時，視為區塊結束邊界，上一個持股區塊必須在此停止，不得延續把其他表格內容誤判為持股。
- 股票名稱空白時不得建立有效持股；同一區塊內連續兩列代碼與名稱皆空白視為區塊已結束，停止該區塊掃描（2026-07-11 新增）。
- 預設股名儲存格填滿色為 `#92D050` 時略過。
- 列內出現設定的文字標記時略過。
- 用股票代碼比對技術資料，不以股票名稱作唯一鍵。

## 股票代碼辨識與正規化（2026-07-11 新增）

- 不得僅靠字串長度（例如「4～10 碼英數字」）判斷股票代碼，該規則會把 Excel 中的金額、權益數等 7～8 位數數字（例如 `10037677`）誤判為股票代碼。
- 股票代碼格式依台股實際商品規則分類（`StockIdentityResolver`）：
  - 4 碼數字：一般上市／上櫃／興櫃股票，或 `00` 開頭之 ETF（`0050`、`0056`）。
  - `00` 開頭 5 碼數字＋選用單一英文字尾：ETF（無字尾）、槓桿反向 ETF（`L`／`R`）、債券 ETF（`B`）、主動式 ETF（其他單一字母，例如 `00982A`）。
  - `91` 開頭 6 碼數字：存託憑證 DR。
  - 其餘 6 碼數字：權證，格式有效但**明確排除於均線策略之外**（短期衍生性商品，價格行為與長期均線策略無關，非程式錯誤），Excel 輸出標示「非策略商品」及原因。
  - 其餘一律視為格式無效，不得判斷為股票代碼，Excel 輸出標示「股票代碼無法識別」。
- 前導零回復（例如 Excel 把 `0050` 讀成數字 `50`）不得盲目補零：僅對 1～3 碼純數字候選嘗試補零為 4／5 碼，且補零後的代碼必須確實存在於官方股票主檔（今日 TWSE／TPEx／TPEx興櫃行情中已出現）才會採用；查無對應候選時仍標示為無法識別（`StockIdentityResolutionService`）。
- 正規化（去除空白、轉大寫）全系統統一由 `StockCodeNormalizer` 處理，Excel、TWSE、TPEx、TPEx興櫃、SQLite、均線計算、策略比對、Excel 輸出皆委派同一套規則，不得各自重複實作。

## 每日五日均價策略前置資料（2026-07-11 修正）

`每日五日均價策略` 的均價資料本身是純 DB 前置作業，與客戶持股比對完全分開。每日官方收盤價寫入 `StockDailyPrice` 後，系統以當日 DB 已保存官方收盤價的股票全集計算並保存 `StockMovingAverage`，欄位只包含：

```text
代碼、名稱、收盤價、5日均價、20日均價、60日均價、120日均價
```

這個前置作業不得讀取或依賴任何客戶頁籤、Excel「現價」、DDE 狀態或 `StrategyAlert`，也不得在每日 Job 中為了補齊 MA120 自動往前回補歷史收盤價。若 DB 現有歷史不足，直接保存 `CalculationStatus=InsufficientHistory`、不足的 MA 欄位為 `NULL`，並在 WinForms 結果頁的「均價資料異常」頁籤顯示該股票與缺少原因；沒有異常時只顯示無異常提示。客戶比對是下游消費者，只能讀取已保存於 DB 的均價資料再與 Excel「進場價/平均價」「現價」比較，不得回頭影響均價保存或換算。

## 策略

> **歷史規則（皆已作廢，僅保留供追溯，不得再套用）**
> - 2026-07-11 舊版：比較基準曾一度改為單一欄位「現價」：`MA5/MA20/MA120 <= 現價`。
> - 2026-07-12 舊版：曾記載每一條均價都必須「進場價/平均價」與「現價」同時大於或等於該均價才成立。
> - 2026-07-13 舊版：MA5／MA20／MA120 任一均價只要 `均價 >= 進場價/平均價` 或 `均價 >= 現價`（OR）即通知。
> 以上「MA5／MA20／MA120 任一均價與進場價/平均價或現價比較」規則，**已由 2026-07-19 新複合條件取代**。

**2026-07-19 現行唯一正式規則**：客戶 Excel 持股表同時有「進場價/平均價」與「現價」兩個完全獨立的欄位，禁止混用、禁止互相代替。唯一的通知條件是下列**複合條件**，且**必須兩項同時成立**才產生通知：

```text
進場價/平均價 > MA20
AND
現價 < MA5
```

程式語意必須明確等同：

```csharp
entryAveragePrice > movingAverage20
&& currentPrice < movingAverage5
```

- **邊界一律嚴格大於／小於**：`進場價/平均價 == MA20` 不觸發；`現價 == MA5` 不觸發。禁止自行改成 `>=` 或 `<=`。
- 只有 `進場價/平均價 > MA20` 單獨成立、或只有 `現價 < MA5` 單獨成立，都**不得**觸發；整體結果只能是兩項同時成立。
- **MA60、MA120 只保存及顯示，不參與觸發**；不得再因 MA120 單獨成立而通知，也不得再採用任一均線成立即通知的 OR 邏輯。
- 通知判斷前，`進場價/平均價`、`現價`、`MA5`、`MA20` 四個數值都必須有效；任一無效或缺少都不得產生「符合策略」通知。
  - `MA5` 缺少：不得判斷，明確顯示缺少 MA5；`MA20` 缺少：不得判斷，明確顯示缺少 MA20；任一缺少即使另一條件成立也不得觸發，且不得用 MA60／MA120 或其他均線替代。保留實際 `CalculationStatus`、`MissingReason`、有效交易日數及最新資料日期。

資料模型欄位沿用但語意已重新定義：`TriggeredMa5`＝「現價 < MA5」子條件、`TriggeredMa20`＝「進場價/平均價 > MA20」子條件、`TriggeredMa120` 固定為 false；整體觸發＝`TriggeredMa5 && TriggeredMa20`。所有 Controller、Service、ViewModel、WinForms 畫面、Excel 輸出及測試都不得再把這些欄位用 OR 組合。

觸發通知需列出客戶、頁籤、股票代碼、股名、進場價/平均價、現價、收盤價、MA5、MA20、MA120、實際兩個子條件與數值，例如：

```text
符合通知條件：
進場價/平均價 120.00 > MA20 115.50；
現價 98.00 < MA5 100.20。
```

未觸發時應明確說明兩個子條件各自是否成立，並強調整體必須兩項同時成立。

### 「進場價/平均價」與「現價」欄位例外規則

「現價」串接外部 DDE（看盤軟體）；「進場價/平均價」**不是** DDE 欄位，是一般手動或公式輸入的成本價。兩者是完全獨立、不得混用的欄位，各自都不可假設一定是正常數字：

- 任一欄位的儲存格為 Excel 錯誤值（現價常見 `#N/A`、`#NAME?`、`#GETTING_DATA` 等）、空白、0、負數或無法解析的文字時，該欄位本身無效；只要任一欄位無效，該持股就不得判斷是否觸發。
- 不得以收盤價、昨日現價、對方欄位的數值或任意預設值代替任一價格本身。
- 必須分別列入「進場價／平均價異常」或「現價異常」清單並寫入 Log，明確告知原因（現價例如「儲存格為 #N/A（DDE 尚未取得資料，看盤軟體可能未開啟或未連線）」；進場價/平均價例如「儲存格為空白，無法讀取進場價/平均價」，且不得寫成 DDE 異常）。兩個欄位同時無效時，兩個原因都必須讓使用者看見，不得只顯示其中一個而隱藏另一個。
- 注意 Excel 錯誤值經 COM 封送為 `0x800Axxxx` 的 Int32，不可與正常整數價格（例如 2042 元）混淆。
- MA5／MA20／MA60／MA120 是本系統依 TWSE／TPEx 官方收盤價自行計算並先保存於 DB，與客戶 Excel 兩個價格欄位是否有效無關；價格異常只代表下游客戶比對無法判斷是否觸發，不得影響均價前置資料的保存與換算。
- 2026-07-11 新增（雙價格異常隔離最高優先規則）：進場價/平均價異常或 DDE 現價異常（`#N/A`／`#NAME?`／`#GETTING_DATA`／`#CONNECT!`／`#BLOCKED!`／空白／0／負數／無法解析文字／看盤軟體未開啟／DDE 尚未連線）只能造成對應欄位「狀態：無效」「最終策略比較：無法判斷」，絕對不可以造成提前 `return`、跳過該股票官方收盤價處理、跳過歷史回補、不計算均線、把已算出的均線改成 `null`、把整批 Job 標記失敗、阻擋其他客戶或其他股票、用收盤價或昨日價格冒充任一價格、用另一欄位的數字代替。必須明確區分「Excel 活頁簿完全無法存取」（工作層級錯誤，可能需要中止）與「活頁簿可存取但某一價格儲存格無效」（僅單筆診斷，不得中斷工作）。

## 全部持股計算結果（2026-07-11 新增）

「每日五日均價策略」頁籤只保存 7 欄均價前置資料：代碼、名稱、收盤價、5日均價、20日均價、60日均價、120日均價。`StrategyEvaluationService.EvaluateAll` 仍可在程式內產生下游客戶比對結果供中央視窗、通知與 Log 使用，但客戶、來源頁籤、Excel 原始列號、市場別、進場價/平均價、現價、DDE 狀態、DDE 錯誤原因、MA 比較、整體結果、觸發條件、鉅亨驗證狀態與計算時間不得寫入此頁籤。`StrategyAlert` 保留不變，只負責中央通知與需要提醒使用者的子集合（觸發、進場價/平均價無效、現價無效、無法判斷），兩者職責不得混用。

## 顯示與寫入

- 成功或失敗都在螢幕正中央顯示。
- 成功畫面分成「均價資料異常」、「符合通知條件（進場價/平均價 > MA20 且 現價 < MA5）」、「進場價／平均價異常」、「現價異常」及「無法判斷」；「均價資料異常」只顯示 `StockMovingAverage.CalculationStatus != Ok` 的 DB 前置資料異常，不含客戶、進場價/平均價、現價或 DDE 欄位。
- 畫面上方摘要必須分開顯示「符合通知條件（進場價/平均價 > MA20 且 現價 < MA5）」「均價資料異常」「進場價/平均價異常」「現價異常」「無法判斷」各自筆數，不得只寫模糊的「符合條件」。
- 「符合通知條件」頁籤的「進場價/平均價」與「現價」必須是兩個獨立欄位，不可合併成單一「價格」欄位；判斷明細欄位逐行列出兩個成立的子條件與實際數值，例如「進場價/平均價 120 > MA20 115.5」「現價 98 < MA5 100.2」。MA120 不參與策略，不列入判斷明細。
- 結果覆寫至 `每日五日均價策略`，避免重跑重複追加。
- 寫入前以 Excel `SaveCopyAs` 建立包含目前記憶體狀態的備份；寫入後驗證通知筆數；最後呼叫 `Workbook.Save`。
