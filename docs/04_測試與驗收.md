# 測試與驗收

## 自動測試

- MA5、MA20、MA120 任一成立即通知。
- 三項都不成立時不通知。
- 技術資料缺少時列為無法判斷。
- 綠色 `#92D050` 與文字標記排除。
- 一般黃色持股不可誤排除。
- 固定來源及 13:35 不可被關閉。
- 鉅亨表格完整解析、缺表頭失敗、頁面日期擷取。

### 官方每日收盤價與均線（2026-07-09 新增，共 68 個自動測試全數通過）

- TWSE／TPEx 正常回應解析（含收盤價含逗號、空白、`--`、`N/A` 之解析與略過）。
- TWSE 休市日明確查無資料；TPEx 靜默回傳前一交易日資料時，Service 層可正確偵測日期不符並拒絕寫入。
- HTTP 200 但為錯誤頁／非 JSON／缺少必要欄位時拋出可重試例外，不視為成功。
- 民國年／西元年、斜線格式／緊湊格式互轉。
- 週末不呼叫官方來源直接記錄休市；平日 TWSE 明確休市時，TPEx 日期不符一併記為休市而非「尚未公布」。
- 同一目標日期重跑不重複呼叫官方來源（冪等略過）。
- 歷史回補只查詢 `targetDate` 以前的日期，不得改寫成執行當日。
- MA5／MA20／MA60／MA120 以固定可人工驗算的收盤價驗證；恰好 120／119 個有效交易日之邊界測試；資料不足時對應均線為 `null` 且不誤將不足天數硬算。
- SQLite 層：同日重跑同一股票只更新不新增重複列；批次含不同交易日整批拒絕；交易中途失敗完整回滾不留下半批資料；依交易日期排序並受最大天數限制。
- 少量 Integration Test（`Category=Integration`）驗證 TWSE／TPEx 目前官方端點與欄位仍可使用；無網路環境可用 `dotnet test --filter Category!=Integration` 排除，執行失敗會明確標示為連線問題而非假裝成功。

## Windows 人工驗收

1. Excel 未開啟：應中央顯示失敗，Log 有明確原因，排程可重試。
2. Excel 正在編輯儲存格：短重試後成功，或達上限顯示失敗。
3. Excel 有 Modal 對話框：不得卡死；關閉後重新執行可成功。
4. 活頁簿唯讀／受保護檢視：不得寫入。
5. 輸出頁籤受保護：不得寫入，原檔不得被破壞。
6. 頁面顯示昨日日期：SQLite 正式技術表不得出現今日資料。
7. 同一天重跑：技術資料、持股快照、通知不得重複。
8. 綠色股名列：不得進入判斷清單。
9. 沒有「自持股」標題但表頭完整的區塊：必須納入。
10. 含「出場價」的已出場表格：不得納入。
11. 中央通知：需在螢幕中央，完整顯示客戶、股票及觸發均價。
12. 系統匣：關閉中央視窗後程式仍在右下角運行。
